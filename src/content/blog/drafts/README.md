# Blog Post Drafts

This folder contains content briefs for potential blog posts before they are written and published.

## Brief Format

Each brief should be a Markdown file named with a descriptive slug (e.g., `my-post-idea.md`) and follow this structure:

```markdown
---
title: "Proposed post title"
tags:
  - Tag1
  - Tag2
description: "One or two sentence summary of the post."
status: draft
---

## Target Audience
Who this post is for.

## Core Argument / Angle
The main thesis or unique perspective.

## Key Points
- Point 1
- Point 2
- Point 3

## Sources / Research
- Any links, data, or references to draw from.

## Notes
Any additional context or instructions for the writer.
```

## Publishing

When a brief is ready to publish, move it to `src/content/blog/` and convert it to a full post using the site's standard frontmatter schema.
